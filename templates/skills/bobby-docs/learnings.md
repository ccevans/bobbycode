### Documenting implementation details (seed)
**Pattern:** Docs agent writes documentation that describes internal implementation rather than user-facing behavior and API contracts.
**Fix:** Focus on what users need to know — inputs, outputs, side effects, and examples. Implementation details belong in code comments, not docs.
